Under the hood

Architecture

A lean, modern mobile stack: a React Native client, a serverless Firebase backend, and intelligence that runs on-device from your own data.

The stack

What Sofra is built with

📱

React Native + Expo

One TypeScript codebase for Android (and iOS-ready). Expo SDK 52, React Navigation for flows, React Native Paper for UI.

🔐

Firebase Authentication

Google Sign-In and email/password, managing identity and sessions.

🗄️

Cloud Firestore

A realtime NoSQL database holding households, meals, dishes, and restaurants — protected by server-side security rules.

🖼️

Cloud Storage

Profile photos, with per-user access rules and size/type limits.

⚡

Zustand state

Lightweight in-memory stores (auth, meals, dishes, household, insights, theme, notifications) keep the UI fast and reactive.

🧠

On-device intelligence

Auto-planning, insights, and forgotten-dish detection are computed on the phone from your history — no meal data leaves for processing.

Data model

How data is organised

Everything is scoped under the household, which is the unit of sharing:

CollectionHoldsAccess
users / {uid}Profile, and a preferences sub-docOwner only
households / {id}Name, invite code, members, adminMembers; admin edits
households / {id} / mealsEvery logged meal (date, type, source, dishes, audience, cost)Household members
households / {id} / dishesThe dish libraryHousehold members
households / {id} / restaurantsPlaces, visits, spend, dish ratingsHousehold members
Design principles

Why it's built this way

Serverless & scalable

No servers to run — Firebase scales automatically and keeps operating costs low as the family base grows.

Reads, minimised

A caching layer avoids redundant database reads when moving between screens, cutting cost and keeping things snappy.

Privacy by structure

Household scoping and deny-by-default rules mean data isolation is enforced by the backend, not just the UI.

Tested foundations

Core logic — planning, date ranges, formatting — is covered by an automated test suite.
